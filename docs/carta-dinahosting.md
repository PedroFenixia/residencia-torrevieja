## Plan de migración completo a OVH

### Fase 1: Email (ANTES de tocar el dominio)
1. **Contratar MX Plan en OVH** (~5€/mes o incluido con hosting) → ovh.es
2. **Crear las cuentas de correo** en OVH (residencia@residenciamarbella.com y las que haya)
3. **Migrar emails antiguos** de Tecnoworld a OVH (con herramienta OVH Mail Migrator o IMAP a IMAP)
4. **Verificar** que el correo funciona en OVH
5. **Configurar el email en los dispositivos** (móvil, ordenador) con los nuevos datos de OVH

### Fase 2: Dominio
1. **Solicitar código EPP** a Dinahosting (Carta 2)
2. **Iniciar transferencia** en OVH: ovh.es/dominios → "Transferir un dominio" → pegar código EPP
3. **Pagar** (~10-12€, incluye 1 año de renovación)
4. **Confirmar** email de verificación
5. Transferencia tarda **5-7 días**

### Fase 3: DNS (una vez dominio en OVH)
Configurar en la zona DNS de OVH:

```
# Web (GitHub Pages)
Tipo A     →  @    →  185.199.108.153
Tipo A     →  @    →  185.199.109.153
Tipo A     →  @    →  185.199.110.153
Tipo A     →  @    →  185.199.111.153
Tipo CNAME →  www  →  pedrofenixia.github.io

# Correo (OVH MX Plan)
Tipo MX    →  @    →  1 mx1.mail.ovh.net.
Tipo MX    →  @    →  5 mx2.mail.ovh.net.
Tipo MX    →  @    →  10 mx3.mail.ovh.net.
Tipo TXT   →  @    →  "v=spf1 include:mx.ovh.com ~all"
```

### Fase 4: Cancelar proveedores antiguos
1. **Cancelar Publicidad Digital e-online AG SL** (Carta 1) — dejar de pagar mantenimiento web
2. **Cancelar Tecnoworld** — una vez email migrado y DNS en OVH
3. **No renovar en Dinahosting** — el dominio ya estará en OVH
